    sequenceDiagram
    participant C as Client
    participant AF as Azure Function

    C->>AF: HTTP Request
    AF->>FA: AsgiMiddleware
    FA->>M: Process middleware
    M->>R: Route to endpoint

    alt Endpoint protegido
        R->>S: Validate JWT
        S->>S: decode_jwt()
        alt Token inv√°lido
            S-->>C: 401 Unauthorized
        end
    end

    R->>BL: Execute business logic
    BL->>ORM: Query data
    ORM->>DB: SQL Query
    DB-->>ORM: Result set
    ORM-->>BL: Python objects
    BL-->>R: Processed data
    R-->>FA: JSONResponse
    FA-->>AF: ASGI response
    AF-->>C: HTTP Response

    Ventajas Implementadas

    ‚úÖ Stateless - Sin estado en memoria - Todo persiste en base de datos - F√°cil escalado horizontal

    ‚úÖ Auto-scaling - Azure Functions escala autom√°ticamente - Basado en demanda de requests - Sin configuraci√≥n manual

    ‚úÖ Connection Pooling - SQLAlchemy maneja pool de conexiones - Reutilizaci√≥n eficiente de conexiones - Reducci√≥n de overhead

    ‚úÖ MQTT Async - Conexiones MQTT son r√°pidas - Desconexi√≥n inmediata post-publicaci√≥n - No bloquea recursos

    Mejoras Sugeridas

    ‚ö†Ô∏è Rate Limiting - No implementado actualmente - Considerar para producci√≥n - Prevenir abuso de API

    ‚ö†Ô∏è Caching - No implementado - Considerar Redis para queries frecuentes - Reducir carga en base de datos
